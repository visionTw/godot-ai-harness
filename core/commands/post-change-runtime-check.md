# /post-change-runtime-check

执行改动后的通用运行闭环检查。

## 适用场景

- `scripts/scenes/resources` 改动后
- 输入链路、窗口交互、布局逻辑变更后
- 提交前最终自检

## 执行清单

1. 启动检查：项目可正常启动。
2. smoke 检查：关键路径可通过。
3. 汇总日志：错误、告警、性能异常。
4. 输出结论：`PASS|CONCERNS|FAIL`。
5. 回写证据到项目文档。

## 输出模板

- 范围：`改动文件 + 功能点`
- 命令：`执行过的命令列表`
- 结果：`PASS|CONCERNS|FAIL`
- 阻断项：`有/无`
- 下一步：`继续提交|先修复`
