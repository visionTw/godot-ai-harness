# Iteration Close Loop（通用）

用于将一次“功能改动”收敛为可验收结果：运行验证、证据沉淀、经验提炼、同步到 harness。

## 触发条件

- 完成功能开发后准备提交。
- 处理完线上/本地问题修复后准备回归。
- 需要把项目内经验抽象成可复用规则时。

## 输入

- 改动范围（文件列表 + 功能点）
- 目标平台（桌面/Windows/macOS）
- 当前阶段或里程碑上下文

## 执行步骤

1. 执行运行验证（启动 + smoke；必要时加打包检查）。
2. 汇总证据：命令、结果、关键告警、是否阻断。
3. 分类经验：
   - 项目专有：保留在业务仓库
   - 通用可复用：同步到 harness 的 rules/skills/commands
4. 输出结论：PASS / CONCERNS / FAIL。
5. 形成提交建议：先 harness，后项目仓库。

## 输出

- 验证报告（可读摘要）
- 可复用条目清单（规则/技能/命令）
- 双仓库提交顺序建议
